We use application Hungarian notation, with these abbreviations:
  • b -- a computed value (type byte)
  • c -- a count of items (type int)
  • d -- an offset into an array or Vector (type int)
  • i -- any other type of index (type int).
  • o -- a flag -- true or false (type boolean)
  • rg -- an array (type in following prefix). We collapse two or more 'rg' sequences into a single 'rgx'.
  • sz -- a string (type String).
  • v -- a vector (type Vector).
  • w -- a computed value (int)
In addition the variable names i, j, k, etc. are used for indices.

All fields should be referenced by being preceded with 'this.'. We don't use a special prefix like _ or m before field names. Contant fields are are not preceded by 'this.' and also do not have a Hungarian notation prefix. They begin with a capital letter.

Methods are named following the usual Java conventions. Classes each specify their Hungarian prefix in their header comments.

Whenever there are several items of a similar purpose (methods, fields, constants, etc.) they are listed in alphabetical order.
